# Padrões de Commit (Conventional Commits)

Use prefixos para categorizar suas mudanças e manter o histórico limpo.

| Prefixo | Significado | Exemplo |
| :--- | :--- | :--- |
| **feat** | Nova funcionalidade | `feat: adiciona filtro de dados` |
| **fix** | Correção de erro | `fix: corrige erro na soma` |
| **docs** | Documentação | `docs: atualiza readme` |
| **style** | Formatação (espaços, etc) | `style: formata codigo com black` |
| **refactor** | Melhoria de código (sem mudar função) | `refactor: simplifica loop for` |
| **chore** | Configurações/Manutenção | `chore: atualiza .gitignore` |
